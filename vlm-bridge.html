VLM Bridge：让 DeepSeek / Kimi 看懂图片

纯文本模型本来看不懂图片，装了这个 Skill 之后，截图拖进去 AI 就能看图说话了

🚀 最快安装方式（推荐）

下载下面的 .md 文件，先准备好一个视觉模型 API Key（推荐 ModelScope，免费注册即得），然后把文件发给 Claude Code 说：

帮我按照这份文档配置 VLM Bridge，我的 API Key 是 ms-xxxxx

Claude Code 会自动完成全部配置。配完后必须重启 Claude Code。

📥

📖 也可以手动安装

下面是从零开始的手把手教程，每一步都有截图和代码可复制。全程大约 15 分钟。

0

开始之前：你需要准备什么

准备工作清单

1. 电脑上装了 Python（命令行输入 python --version 检查）

2. 一个视觉模型的 API Key（推荐 ModelScope，免费额度；也支持 OpenAI、SiliconFlow）

3. 大概 15 分钟时间

Windows 用户不需要额外装任何东西。

1

第一步：创建文件夹

打开终端（Windows 按 Win+R 输入 powershell，Mac 打开 Terminal），复制下面这行命令回车：

bash mkdir -p ~/.claude/vlm-bridge/.claude/hooks

没有任何输出就是成功了。

2

第二步：保存脚本文件

用记事本新建文件，把以下脚本全选复制进去：

3

第三步：获取 API Key（三选一）

方案 A：ModelScope（推荐，有免费额度）

1. 打开 modelscope.cn，注册并登录

2. 进入「设置 → 账号绑定」→ 绑定阿里云账号（很多人漏掉这步！）

3. 进入「设置 → API Key」→ 创建新 Key，复制 ms- 开头的字符串

4. 把脚本里的 YOUR_API_KEY_HERE 替换为这个 Key

方案 B：SiliconFlow（国内直达，也有免费额度）

注册 siliconflow.cn，获取 Key 后改脚本里两处：

python "api_base": "https://api.siliconflow.cn/v1", "api_key": "你的SiliconFlow Key", "model": "Qwen/Qwen2-VL-72B-Instruct",

方案 C：OpenAI

python "api_base": "https://api.openai.com/v1", "api_key": "sk-你的OpenAI Key", "model": "gpt-4o",
4

第四步：注册 Hook（让 Claude Code 知道你装了这个脚本）

这一步是干什么的？

你刚才保存的脚本，Claude Code 并不知道它的存在。Hook 就是告诉 Claude Code：「每次发消息之前，先跑一下这个脚本」。注册 Hook 就是填一个配置文件。

怎么操作：

情况一：你从来没改过 settings.json（大多数人是这个情况）

直接把下面的内容全选复制，粘贴到下面这个文件里（文件不存在就新建）：

文件路径 C:\Users\你的用户名\.claude\settings.json

怎么打开这个文件：打开记事本 → 文件 → 打开 → 在地址栏粘贴上面这个路径 → 回车。如果提示文件不存在，就点「新建」。

▶ Windows 用户：复制下面这整段
json { "hooks": { "UserPromptSubmit": [ { "hooks": [ { "type": "command", "command": "python ~/.claude/vlm-bridge/.claude/hooks/image_preprocess.py", "timeout": 180 } ] } ] } }

注意：命令里写的是 python，不是 python3。

▶ Mac / Linux 用户：复制下面这整段
json { "hooks": { "UserPromptSubmit": [ { "hooks": [ { "type": "command", "command": "python3 ~/.claude/vlm-bridge/.claude/hooks/image_preprocess.py", "timeout": 180 } ] } ] } }

还要给脚本加执行权限，终端里运行：

bash chmod +x ~/.claude/vlm-bridge/.claude/hooks/image_preprocess.py

情况二：你的 settings.json 里已经有其他配置了

那就只加 "hooks" 这一块，加到已有内容的大括号里面。不会操作的话，直接把文件内容复制发给 Claude Code，说「帮我把 hook 配置加进去」，它会帮你改。

5

第五步：重启并测试

必须重启 Claude Code——关掉终端重开，Hook 才会生效。

测试：截一张图拖进 Claude Code → 输入「帮我看看这张图」→ 立刻发送（不要等超过 2 分钟）。

如果 AI 回复中出现了 [Current turn image analysis ...] 开头的文字描述，就说明成功了。

6

常见问题排查

拖入图片后没任何反应？
检查：① 重启 Claude Code 了吗？② settings.json 路径写对了吗？
回复里有 [Image #1] 但没有分析文字？
拖入图片后等太久才发送。缓存只有 120 秒，拖入后立即发送。
终端报 python3: command not found？
Windows 用户 settings.json 里要用 python，不要用 python3。
终端报 401 Unauthorized？
Key 写错，或者 ModelScope 没绑阿里云账号。去 modelscope.cn 绑定。
想换别的视觉模型？
改脚本 CONFIG 里的 api_base、api_key、model 即可。
💡

原理（一句话）

拖入图片 → Claude Code 缓存图片生成 [Image #N] → Hook 把图片发给视觉模型分析 → 文字描述塞进消息 → 纯文本模型基于描述回答。

只分析当前这轮的图片，不会把旧截图混进来。
