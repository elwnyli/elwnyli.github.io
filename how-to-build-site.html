开始之前：你需要准备什么

你不需要会写代码，不需要懂 HTML/CSS，不需要懂服务器。你只需要：

准备工作清单

1. 一台电脑（Windows 或 Mac 都行）

2. 安装了 Claude Code（如果还没装，先看本站的《Claude Code 安装教程》）

3. 一个 GitHub 或 Gitee 账号（注册免费，几分钟搞定）

4. 耐心——过程中可能会遇到网络问题，别急，一个个解决

1

第一步：跟 Claude Code 说出你的想法

这是最关键的一步。你不需要懂技术术语，用大白话说就行。

我当时的原话就一句话：

我实际用的提示词
我想要一个自己的网站，然后第一个内容就是分享 Claude Code 安装教程

就这一句话，Claude Code 就开始帮我干活了。它会追问你几个问题，你像跟朋友聊天一样回答就行：

Claude Code 问：网站叫什么名字？
我答：叫「土豆熬孬土豆」（随便起的，后面可以改）
Claude Code 问：想要什么视觉风格？
我答：暖色雅致

几秒钟后，它就帮我生成了完整的两页网站：一个首页 + 一篇教程文章，配色、字体、排版全都弄好了。

关键技巧

描述网站时可以说这些方面：网站用途（分享教程/写博客/展示作品）、喜欢的颜色、参考某个你喜欢的网站风格。

不确定怎么说？直接说「我要一个简单干净的网站放教程」，Claude Code 会自己补全细节。

第二步：预览你的网站

代码写好了，怎么看效果？直接问：

你只需要说
我想看看网页效果 / 怎么在浏览器里打开我的网页

Claude Code 会帮你安装 VS Code（免费的代码编辑器）和 Live Server 插件。装好之后，右键 HTML 文件 → Open with Live Server，浏览器就会自动打开你的网站。

从此以后，你在 VS Code 里改完代码 Ctrl+S 保存，浏览器自动刷新预览，不用手动操作。

第三步：修改内容（不需要懂代码）

打开 HTML 文件后，你会看到一堆英文标签和中文文字混在一起。你只需要关注中文文字部分，直接改，像改 Word 一样。

改完按 Ctrl+S 保存，浏览器自动刷新就能看到效果。改错了按 Ctrl+Z 撤回。

我的经验

改标题 → 找 h1 附近的中文

改正文 → 找 p 附近的中文

如果找不到要改的文字在哪，直接问 Claude Code：「帮我把 xxx 这句话改成 yyy」，它会精准告诉你在第几行。

第四步：添加新文章

当你想发第二篇教程时，不需要从头写代码。直接问：

你只需要说
我想要添加新文章，但不想自己写代码，帮我做一个空白模板

Claude Code 会创建一个 template.html。以后加新文章就三步：

1. 复制 template.html，重命名（比如 new-post.html）

2. 打开文件，Ctrl+F 搜索 【（中文方括号），把所有占位符替换成你的内容

3. 在 index.html 里复制一张文章卡片，把链接改成你的新文件名

第五步：让全世界看到你的网站（部署）

现在网站只是你电脑上的本地文件，别人看不到。需要「部署」到互联网。

你只需要说
帮我部署到网上 / 我想让其他人也能访问我的网站

以下是我实际经历的完整过程——

方案一：GitHub Pages —— 失败了

Claude Code 首先尝试了 GitHub Pages。但因为网络问题（GitHub 在国内被墙），连不上，放弃了。

方案二：Gitee Pages —— 也失败了

代码成功推到了 Gitee（码云，国内版 GitHub），但在后台找了很久，Gitee Pages 入口藏得太深，而且需要实名认证。

📷 待配图Gitee 仓库界面，红圈标出「服务」菜单和里面找不到 Pages 的情况

方案三：Netlify —— 成功了…一段时间

Claude Code 用 Netlify 部署成功。拖拽上传就行，很方便。具体过程：

1. Claude Code 用命令行把网站文件打包上传到 Netlify

2. 获得了一个临时网址

3. 我打开认领链接，用 GitHub 账号登录 Netlify

4. 在 Netlify 后台把网站名从随机字符改成了 elwynlee

5. 网站顺利上线：elwynlee.netlify.app

方案四：Netlify 额度用完 → 迁到 Vercel

使用了一段时间后，因为我频繁更新网站（一天部署好几次），Netlify 的免费额度用完了，部署被冻结。

于是 Claude Code 帮我换到了 Vercel，同样免费，额度更宽松：

1. Claude Code 帮我登录 Vercel，用命令 `npx vercel --prod --yes` 一键部署

2. 获得新网址：elwynlee-site.vercel.app

3. 还帮我创建了一个 `更新网站.bat` 文件

📷 待配图Vercel 部署成功画面
最终经验

经历了四个方案才最终稳定下来：GitHub（被墙）→ Gitee（找不到 Pages）→ Netlify（额度用完）→ Vercel（目前最佳）。

重点不是哪个平台最好，而是Claude Code 会帮你不断尝试。你不需要自己研究这些平台，告诉它「帮我部署」就行。

6

第六步：日常更新网站

网站上线后，日常更新超级简单：

1. 用 VS Code 打开要改的 HTML 文件，改文字

2. Ctrl+S 保存

3. 双击「更新网站.bat」（Claude Code 帮你建好的）

4. 等几秒，网站更新完成

就这么简单

不需要打开任何网页、不需要登录、不需要拖拽。改完 → 保存 → 双击 bat → 搞定。

我遇到过的坑

GitHub 连不上怎么办？
让 Claude Code 换 Gitee。Gitee 不行换 Netlify，Netlify 不行换 Vercel。总有一个能用的。
图片拖到网站里不显示？
确认图片文件（.png 或 .jpg）和 .html 文件在同一个文件夹里。
改完代码网站没变化？
检查三件事：Ctrl+S 保存了没？浏览器刷新了没（按 F5）？双击「更新网站.bat」推上线了没？
部署后网站还是旧版？
等几十秒，CDN 缓存刷新需要时间。如果一分钟后还是旧的，再双击一次「更新网站.bat」。
HTML 文件直接双击打开显示乱码？
让 Claude Code 给你加个 UTF-8 BOM 头。它会自动修复。
部署平台额度用完了怎么办？
告诉 Claude Code「我的网站部署额度用完了，帮我换个平台」，它会帮你迁移。我们就是从 Netlify 迁到 Vercel 的。
9

总结：你需要的全部技能

三个技能就够：

1. 会打字，能用大白话跟 Claude Code 描述你想要的

2. 不慌，遇到报错把错误信息原样复制给 Claude Code

3. 愿意试，Claude Code 提供的不同方案都试试，总有一个能用

一句话总结

你不需要会写代码、不需要懂 HTML/CSS、不需要懂服务器。你只需要会提问。
